5. How Custom Variables SHOULD Work (Like br_norm!)

üí° THE SOLUTION: Custom variables should work EXACTLY like br_norm!
They should handle their own dependencies in their evaluation method.

br_norm Pattern

@property def br_norm(self): if needs_calculation: self._calculate_br_norm() return self._br_norm_manager def _calculate_br_norm(self): trange = self._current_operation_trange # FETCH DEPENDENCIES! get_data(trange, proton.sun_dist_rsun) # CALCULATE! br_norm = self.raw_data['br'] * ... self.raw_data['br_norm'] = br_norm

Custom Variable Pattern (SHOULD BE)

def evaluate(self, name, trange): """Evaluate custom variable""" # Step 1: GET SOURCE VARIABLES! sources = self.get_source_variables(name) get_data(trange, *sources) # ‚Üê RECURSIVE! # Step 2: EVALUATE LAMBDA! result = self.callables[name]() # Step 3: STORE RESULT! self.variables[name] = result return result

7. Why This Works

The Recursive Pattern:

  1. plotbot_main calls get_data(trange, abs_bn)
  2. get_data sees custom_data_type, asks container to evaluate
  3. container.evaluate() calls get_data(trange, mag_rtn_4sa.bn)
  4. get_data loads mag_rtn_4sa (normal CDF download)
  5. container.evaluate() continues, evaluates lambda
  6. get_data marks custom variable as complete
  7. plotbot_main plots everything
‚úÖ NO PHASES - Just recursive dependency resolution!
‚úÖ NO COMPLEXITY - Each variable knows how to load itself!
‚úÖ SAME PATTERN - br_norm and custom variables work identically!

üéØ The Solution

Move dependency loading INTO custom_variables.evaluate()

Just like br_norm handles its dependencies!

  • ‚úÖ Simple handler in get_data.py (10 lines)
  • ‚úÖ evaluate() loads sources (using get_data recursively)
  • ‚úÖ plotbot_main stays simple (no phases!)
  • ‚úÖ Works exactly like br_norm pattern
